# 冰小说创作技能 (ice-novel-skill)

**辅助中文网文长篇创作的 Claude Code Skill。**

核心设计目标：**低上下文消耗、人工可控、一致性保障**。

- **低上下文消耗** — 通过 `[[文件链接]]`（Obsidian 风格）按需加载文件，避免无关信息占用上下文
- **人工可控** — 逐幕生成、逐幕校对，AI 不擅自推进剧情
- **一致性保障** — 角色三维演变追踪 + 伏笔追踪 + 三级检查体系

---

## 安装

将 command 文件复制到 Claude Code 的 commands 目录即可：

```bash
cp .claude/commands/*.md ~/.claude/commands/
```

本 skill 的命令不依赖仓库内 `templates/` 目录；`/开书` 会直接生成标准初始化文件。仓库里的 `templates/` 仅作为参考样例。

---

## 快速开始

```bash
# 在 Claude Code 中（项目目录下）：
/开书 --project-root .
```

按引导完成创意孵化 → 设定构建 → 大纲拆解 → 开始写作。

## 完整工作流

```text
提出 idea
  ↓
/开书 — 确定核心创意，初始化项目结构
  ↓
/定设定 — 构建世界观 + 人物小传
  ↓
/拆大纲 — 卷→篇→章三级拆解（含角色弧光规划 + 伏笔规划）
  ↓
确认风格指南（style-guide.md）— 必须在开始写之前完成
  ↓
/写章 — AI 生成章前简报 → 用户确认 → 逐幕生成 → 章级检查
  ↓
/检查 — 章级 → 篇级 → 卷级检查
```

### 各阶段产出物

| 阶段 | 产出物 | 人工介入 |
|------|--------|---------|
| 1. 创意孵化 | `idea.md` | 确认核心创意 |
| 2. 世界观构建 | `world.md`, `characters/*.md`, `settings/*.md` | 逐项审核设定 |
| 3. 大纲拆解 + 弧光规划 | `outline.md`（含弧光 + 伏笔规划） | 确认结构与角色走向 |
| 4. 风格确定 | `style-guide.md`（正反面示范） | 确认风格锚点 |
| 5. 章节写作 | `draft/volX/partY/chZ.md`（每章 3～5 幕） | 每幕校对 |
| 6. 层级检查 | `reports/` 下各级检查报告 | 确认检查结论 |

## 核心设计

### Obsidian 链接机制

所有文件间引用使用 `[[filename]]` 语法。AI 生成时按需加载：

- **起点**：outline 当前章表格
- **第 1 层**：起点中所有 `[[链接]]` 的文件
- **第 2 层**：第 1 层文件中出现的 `[[链接]]`
- **停止**：第 3 层及更深不追踪（用户可手动覆盖）

### 逐幕写作模式

每章 2000～4000 字，拆为 3～5 幕。交互流程：

```
/写章 → 生成章前简报（含幕结构建议）→ 用户确认
  → AI 写第 1 幕 → 用户校对确认
  → AI 写第 2 幕 → 用户校对确认
  → ...
  → 全部幕完成 → 自动章级检查
```

### 角色三维演变

每个角色追踪三个维度的变化：

- **能力演变** — 每次能力提升/获得/丧失，标注触发事件
- **性格演变** — 核心性格内核不变，外在表现渐变
- **心态演变** — 对关键问题的态度变化，与剧情挂钩

### 伏笔追踪（三态流转）

```
计划中 → 已埋设 → 已回收
```

两表互补：`outline.md`（伏笔规划总表，静态） + `foreshadow.md`（运行状态，动态）。

### 三级检查体系

| 层级 | 检查频次 | 覆盖范围 |
|------|---------|---------|
| 章级 | 每章 | 一致性、节奏、风格、伏笔 |
| 篇级 | 每篇 | 结构、连续性、风格一致性 |
| 卷级 | 每卷 | 主线推进、角色弧光、伏笔完整性 |

---

## 设计理念

1. **不赶进度** — 逐幕校对保障质量，AI 不擅自推进
2. **不浪费上下文** — `[[链接]]` 按需加载，无关文件不读
3. **不丢失线索** — 伏笔表和角色演变表追踪全程
4. **不依赖神奇数字** — 节奏用档位（慢/中/快）而非字数硬约束
