## 准备工作

在 Claude Code 中加载本 skill：

```bash
# 进入你的小说项目目录（新建或已有）
mkdir my-novel
cd my-novel

# 确认 skill 已安装
# ice-novel-skill 的 command 文件应在 ~/.claude/commands/ 下
# 如未安装，将 ice-novel-skill/.claude/commands/ 复制到 ~/.claude/commands/
```

## 第一步：开书

```bash
/开书 --project-root .
```

AI 会引导你完成「创意孵化」对话，逐步确认：

1. 故事核心（一句话概括）
2. 主角设定
3. 世界基调
4. 目标读者和平台

完成后自动创建项目目录结构和模板文件。

> **注意**：`/开书` 用对话引导，不必一次想全。你可以说"还不确定，先记个大概"，后面随时改 `idea.md`。

---

## 第二步：定设定

```bash
/定设定 --project-root .
```

AI 逐项引导你确认世界观设定：

**必填**（每项 AI 会逐个问你）：
- 世界背景和时代
- 力量体系（如修仙等级、魔法体系）
- 主要势力
- 关键历史事件

**角色设定**（AI 逐个创建角色文件）：
- 主角（`characters/protagonist.md`）
- 反派（`characters/antagonist.md`）
- 重要配角
- 每个角色包含：基本信息、人际关系、对话特征

> **提示**：角色文件的「三维演变表」先空着，写作中自动填充。

---

## 第三步：拆大纲

```bash
/拆大纲 --project-root .
```

这是最关键的步骤。AI 引导你完成：

### 3.1 故事概要 + 核心冲突

填写 `outline.md` 开头：
- 一句话版、一段话版故事概要
- 主线矛盾、内在冲突、最终目标、主要阻碍

### 3.2 规划角色弧光

每个核心角色确定：
- 弧光类型（成长型/堕落型/救赎型/平弧型）
- 三维演变里程碑表（精确到章号）

示例：

```
| 维度 | ch01 起点 | 里程碑1（ch15） | 里程碑2（ch30） | 终点（ch50） |
|------|----------|----------------|----------------|------------|
| 能力 | 练气初期 | 练气中期（宗门大比） | 筑基（秘境传承） | 元婴初期 |
| 性格 | 怯懦逃避 | 承担责任（师父之死） | 主动布局 | 沉稳果决 |
| 心态 | 想回家 | 认同此界 | 有了要保护的人 | 守护此界 |
```

> **规则**：每列必须有具体章号 + 触发事件。弧光表中的章号必须在大纲中体现。

### 3.3 卷→篇→章拆解

AI 引导你逐层拆解：
1. 全书分几卷？（3～10 卷）
2. 每卷分几篇？（3～10 篇）
3. 每篇分几章？（3～10 章）

每章填写 12 个字段：

| 字段 | 填什么 | 为什么重要 |
|------|--------|-----------|
| 剧情摘要 | 2-4 句本章发生什么 | AI 写作时最重要的输入 |
| 节奏档位 | 慢档/中档/快档 | 控制卷级节奏 |
| 视角角色 | 谁在看 | 固定叙事视角 |
| 出场角色 | 谁出场 | 控制人物出场 |
| 本章目标 | 必须完成的事 | 防止 AI 偏航 |
| 弧光推进 | 推哪条线、触发什么 | 让该突出的场景突出 |
| 章末钩子 | 结尾留什么悬念 | 防止章节平平收尾 |
| 禁区 | 绝对不能写什么 | 防止提前泄气 |
| 埋设伏笔 | F0XX | 伏笔系统输入 |
| 回收伏笔 | F0XX | 伏笔系统输出 |
| 情绪走向 | 从情绪A→情绪B | 防止情感扁平 |
| 关联章节 | 承接/铺垫哪章 | 跨章衔接 |

### 3.4 伏笔规划总表

AI 汇总所有伏笔，生成全局 ID（F001、F002...）：

```
| ID | 伏笔内容 | 埋设章 | 回收章 | 关联角色 | 类型 | 备注 |
```

> 这一步完成后，`foreshadow.md` 会自动同步，所有伏笔进入「计划中」状态。

## 第四步：填风格指南

在开始写第一章之前，必须完成 `style-guide.md`。

**至少需要填写**：
- 文风 DNA（叙述者态度、幽默类型、情感描写方式）
- 叙事参数（人称、叙事距离、时态）
- 正面示范：至少 3 个风格锚点（每段原文 + 为什么选 + AI 可学什么）
- 场景类型标杆：至少填 3 种场景类型

**禁止清单已预填**，开书时自动生成。

> **如果不填会怎样**：AI 写出来的文风不可控，可能偏离你的预期。风格审查时频繁标记。

## 第五步：写章

```bash
/写章 --project-root .
```

这是日常写作命令。执行后分三个阶段：

### 阶段一：章前简报

AI 读取 outline 当前章表格，自动生成简报：

```
# 章前简报 — 第 1 章：醒来

## 一、必须加载的文件
| 层级 | 文件 | 加载原因 |
|------|------|---------|
| 起点 | outline.md 当前章 | 核心指令 |
| L1 | [[protagonist]] | 视角角色 |
| L1 | [[mentor]] | 出场角色 |
| L2 | [[power-system]] | 能力关联设定 |

## 二、本章核心指令（从大纲提炼）
- 剧情任务：...
- 角色任务：...
- 禁区：...

## 三、幕结构建议
| 幕 | 功能 | 情绪 | 字数 | 衔接点 |
|----|------|------|------|-------|
| 第1幕 | 场景建立 | 压抑 | 800字 | — |
| 第2幕 | 冲突展开 | 紧张 | 1000字 | 承接第1幕发现的异常 |
| 第3幕 | 章末钩子 | 危机 | 800字 | 承接第2幕试探的结果 |
```

**你可以在确认前**：
- 增删加载的文件（比如某个 L2 文件不重要就删掉）
- 调整幕数（3/4/5 幕）
- 修改幕的功能分配和字数预算
- 补充禁区（比如"这章别让反派露面"）

**确认后进入阶段二**。

### 阶段二：逐幕生成

AI 写一幕，你确认一幕：

```
AI 写第 1 幕（800 字）→ 展示给你
你：确认，继续
AI 写第 2 幕（950 字）→ 展示给你
你：这里节奏慢了，压缩到 700 字
AI 改写第 2 幕（700 字）→ 展示给你
你：可以，继续
AI 写第 3 幕（750 字）→ 展示给你
你：确认
```

**你可以**：
- 说"继续"确认当前幕
- 给具体修改意见（"把这段对话改短""加一句环境描写"）
- 给方向性意见（"这里太赶了""反派应该更有压迫感"）
- 每幕确认后才进入下一幕

**幕间衔接规则**（AI 自动遵守）：
1. 第 N 幕开头必须回应第 N-1 幕结尾的落点
2. 时间/地点跳跃必须有过渡说明
3. 同一章内不换视角（除非简报中批准）
4. 最后一幕就是章末钩子

### 阶段三：章级检查

全部幕完成后，AI 自动执行：

**AI 自动核查**：
| 检查项 | 说明 |
|--------|------|
| 总字数 | 2000～4000 字 |
| 各幕字数偏差 | 是否偏离幕结构建议 >30% |
| 章末钩子 | 末段是否落到钩子 |
| 情绪走向 | 是否符合简报规划 |
| 禁区扫描 | 是否有违规 |
| 最长连续对话 | >15 句标记 |

**需要你确认**：
- 钩子是否有效
- 节奏是否合适
- 高潮/转折力道

检查通过后，AI 自动更新 `foreshadow.md` 和角色演变表。你可以继续写下一章。

## 日常维护

### 查看伏笔

```bash
/伏笔 --project-root . --action view
```

按三区展示：计划中 / 已埋设 / 已回收。可添加 `--action alert` 检查超期伏笔。

### 查看角色状态

```bash
/角色状态 --project-root . --character protagonist
```

展示角色当前能力/性格/心态，并与大纲规划的弧光里程碑对比（正常/延迟/超前）。

### 设定变更

如果中途改了设定（比如把"筑基期"的实力定义改了）：

```bash
/更新设定 --project-root . --file settings/power-system.md --changes "筑基期定义修改"
```

AI 通过反向链接找出所有引用者，逐项检查是否需要同步。

## 检查与修订

### 章级检查

每章完成后自动执行。也可手动运行：

```bash
/检查 --project-root . --level chapter --target ch03
```

### 风格审查

如觉文风不对：

```bash
/风格审查 --project-root . --chapter ch03
```

AI 扫描禁止清单违例、叙述视角漂移、角色对话人设是否一致。不修改正文，只出报告。

### 篇级/卷级检查

写完一篇或一卷后：

```bash
/检查 --project-root . --level part
/检查 --project-root . --level volume
```

## FAQ

### 写到一半发现大纲不合理，能改吗？

可以。直接编辑 `outline.md`，然后：
1. 如果改了伏笔规划，运行 `/伏笔 --project-root . --action sync` 同步
2. 如果改了角色弧光，运行 `/角色状态` 检查当前进度是否需要调整
3. 如果改了设定，运行 `/更新设定` 级联同步

### 章前简报的加载文件列表不对怎么办？

简报生成后、确认前，你可以手动增删。追加文件用 `+`，移除用 `-`。你的调整优先级高于 AI 的 2 层规则。

### AI 写出来的对话不像这个角色？

检查 `characters/<角色>.md` 的「对话特征」节是否完善。完善后重新写章，AI 会自动参考。

### 检查报告中有标记项，必须处理吗？

**AI 标记项**（字数偏差、禁区扫描等）需逐一确认——确实是问题的改正文，不是问题的注明原因即可。

**作者确认项**（钩子有效、节奏等）是你自己的判断，你觉得没问题就通过。

### 伏笔超期了怎么办？

运行 `/伏笔 --project-root . --action alert` 查看超期清单。然后在当前章或下一章提前回收，或者在 outline 中调整回收章号。

### 我能跳过逐幕确认吗？

不能。逐幕校对是技能的核心设计——AI 不擅自推进。如果某幕写得满意，只需要说"继续"即可，不耗时。
