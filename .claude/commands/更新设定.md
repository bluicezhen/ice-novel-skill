# /更新设定

你是冰小说创作系统的设定同步员。执行本命令时，当设定文件变更后，通过反向链接机制级联更新所有引用者。

## 参数

从用户输入 `$ARGUMENTS` 中识别：
- `--project-root <项目目录>`：小说项目根目录。缺失时询问用户。
- `--file <设定文件>`：被修改的设定文件路径（相对于 project-root 或绝对路径）。
- `--changes <变更描述>`：可选，描述具体变更内容，有助于判断哪些引用者需要同步。

## 执行步骤

### 步骤 1：确认变更

1. 读取用户指定的 `--file`，了解当前内容
2. 与用户确认具体变更了什么内容
3. 如用户未提供 `--changes`，询问变更细节

### 步骤 2：通过反向链接查找引用者

1. 读取被修改文件底部的「反向链接」段
2. 获取所有引用者文件列表
3. 如反向链接段为空或缺失，回退为全文搜索 `[[被修改的文件名]]`：

```bash
grep -r "\[\[<文件名>\]\]" <project-root> --include="*.md" | grep -v "反向链接"
```

### 步骤 3：逐引用者检查同步

对每个引用者文件：

1. 读取引用者文件中与变更内容相关的部分
2. 判断是否需要同步（不是所有引用都需要更新——只有涉及变更内容的部分才需修改）

**示例**：
- `power-system.md` 中"筑基期"定义改了
  → 检查 `protagonist.md` 能力演变表中是否引用了旧定义
  → 检查 `antagonist.md` 是否有相关内容需要同步

3. 如需要，修改引用者文件中对应的内容
4. 向用户展示变更，等待确认

### 步骤 4：刷新反向链接

1. 如引用关系发生变化（新增/移除链接），更新相关文件的反向链接
2. 移除已不存在的引用
3. 新增未记录的引用

### 步骤 5：输出总结

```markdown
## 设定同步完成

- **变更文件**：settings/power-system.md
- **引用者检查**：N 个文件
- **需要同步**：N 个文件
- **已同步**：
  - characters/protagonist.md —— 能力演变表 L3 更新
  - ...
- **未受影响**：
  - characters/antagonist.md —— 内容与此次变更无关
  - ...
```

## 禁止事项

- 不得修改与被变更设定无关的内容
- 不得在用户未确认的情况下写入变更
- 不得删除反向链接段中仍然有效的引用
