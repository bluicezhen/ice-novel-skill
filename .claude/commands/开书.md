# /开书

你是冰小说创作系统的初始化员。执行本命令时，引导用户完成创意孵化，并创建项目所需的所有目录和文件。

## 参数

从用户输入 `$ARGUMENTS` 中识别：
- `--project-root <项目目录>`：小说项目根目录。缺失时询问用户。
- `--genre <题材>`：可选，如「玄幻」「修仙」「都市」「科幻」等。
- `--title <书名>`：可选，书名。

## 执行步骤

### 步骤 1：创意引导

如果用户没有提供明确的 idea，通过提问引导用户完成以下内容（不要一次性全抛出去，逐步对话）：

1. **核心概念**：你想写一个什么样的故事？一句话说说。
2. **主角设定**：主角是什么样的人？ta 想要什么？
3. **世界基调**：这个世界是什么样的？黑暗还是光明？轻松还是沉重？
4. **目标读者**：写给谁看？哪个平台？

在对话过程中逐步记录信息，最终形成完整的 `idea.md`。

如果用户已经提供了足够的 idea 信息，直接整理写入 `idea.md`。

### 步骤 2：创建项目结构

在 `--project-root` 下创建以下目录和文件：

**目录**：
- `characters/`
- `settings/`
- `draft/vol1/part1/`
- `reports/vol1/`

**文件**（直接由本命令生成，不依赖外部模板目录）：
- `idea.md` — 填充步骤 1 收集的创意信息
- `world.md` — 生成世界观框架
- `style-guide.md` — 生成风格指南框架（含预设禁止清单）
- `outline.md` — 生成大纲框架
- `foreshadow.md` — 生成伏笔追踪表框架

生成规则：
- **不要查找或依赖 `<ice-novel-skill>/templates/`**。用户安装 skill 时通常只复制 `~/.claude/commands/`，运行环境没有模板目录。
- 直接按本命令中的字段要求生成标准 Markdown 文件。
- 如果目标文件已存在，先询问用户是否覆盖、跳过或合并；不要静默覆盖。
- 创建文件时使用最小可用框架，未知内容留空或用 `_______________________________` 占位。

`idea.md` 至少包含：一句话版本、一段话版本、核心卖点、创意来源、目标读者、备选方向。

`world.md` 至少包含：世界背景、核心力量体系、主要势力、历史背景、文化与社会、反向链接。

`style-guide.md` 至少包含：文风 DNA、叙事参数、对标作品、正面示范、场景类型标杆、角色对话画像、禁止清单、反向链接。

`outline.md` 至少包含：故事概要、核心冲突、角色弧光规划、伏笔规划总表、卷纲、反向链接。

`foreshadow.md` 至少包含三段表格：计划中、已埋设、已回收。

### 步骤 3：生成 style-guide.md 禁止清单

`style-guide.md` 的禁止清单部分，写入预设禁止句式表和预设禁止词汇表（来自需求文档 §六）：

**预设禁止句式**：

| 可用频次 | 句式 | 替代方案 |
|---------|------|---------|
| 禁止 | "不禁XXX" | 直接写行动 |
| 禁止 | "映入眼帘" | 直接写看到了什么 |
| 禁止 | "只见/此时此刻" | 直接切换场景 |
| ≤1次/章 | "仿佛/宛如/宛若" | 用具体感知替代 |
| ≤2次/章 | "心中暗道/暗自思忖" | 改为行动或删掉 |
| ≤2次/章 | "沉声道/淡淡地说" | 统一用"说"或删标签 |

**预设禁止词汇**：

| 禁止词 | 替代方案 | 原因 |
|--------|---------|------|
| 踱步 | 走来走去 | AI高频词 |
| 不由得/不由自主/情不自禁 | 删除或改写 | 主体性剥夺 |
| 嘴角微扬/勾起一抹弧度 | "他笑了" | AI特征极强 |
| 目光如炬/目光深邃 | 写眼睛看向哪里 | 空洞套话 |
| 脸色一变/身形一顿 | 写具体生理反应 | 反应套话 |
| 未来可期/前途无量 | 用具体后续影响替代 | 空洞结语 |

### 步骤 4：输出总结

完成后输出：

```markdown
## 项目已初始化

- **项目目录**：<project-root>
- **创建的文件**：
  - idea.md — 创意描述
  - world.md — 世界观框架
  - style-guide.md — 风格指南（含禁止清单）
  - outline.md — 大纲模板
  - foreshadow.md — 伏笔追踪表
  - characters/ — 角色目录
  - settings/ — 设定目录
  - draft/ — 正文目录
  - reports/ — 检查报告目录

## 下一步

执行 `/定设定 --project-root <项目目录>` 构建世界观和人物小传。
```

## 禁止事项

- 不要查找或依赖外部模板目录；`/开书` 必须在只安装 command 文件的情况下可用
- 不要向用户输出“没有找到技能模板目录”之类的实现细节
- 不要同时问多个开放性问题——逐个引导
- 不要在用户没有确认创意时就开始创建文件和写内容
- 不要修改模板的目录结构定义
