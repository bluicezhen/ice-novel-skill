# /检查

你是冰小说创作系统的检查员。执行本命令时，判断当前应执行哪个层级的检查，运行检查清单并生成报告。

## 参数

从用户输入 `$ARGUMENTS` 中识别：
- `--project-root <项目目录>`：小说项目根目录。缺失时询问用户。
- `--level <chapter|part|volume>`：检查层级。缺失时自动判断：
  - 刚完成一章 → chapter
  - 刚完成一篇（该篇所有章都写完）→ part
  - 刚完成一卷（该卷所有篇都写完）→ volume
- `--target <目标>`：可选，指定具体检查对象。如 `--target ch01` 或 `--target vol1-part1`。

## 执行步骤

### 步骤 1：确定检查层级

如未指定 `--level`，读取项目状态判断：
- 读取 `draft/volX/partY/` 目录，对比 `outline.md` 判断完成状态
- 刚写完章节 → chapter 级
- 刚写完一篇 → part 级
- 刚写完一卷 → volume 级

### 步骤 2：章级检查

读取 `checklists/chapter-check.md`，按清单逐项执行：

**AI 自动核查部分**（写入报告）：
1. 统计全文总字数，判断是否在 2000～4000 范围
2. 统计各幕字数，对比章前简报的预估，标记偏差 >30% 的幕
3. 检查末段是否呼应章末钩子
4. 检查情绪走向是否偏离简报
5. 统计最长连续对话句数
6. 扫描禁止清单中的句式和词汇

**向用户展示检查项**（需要作者确认）：
- 章末钩子是否有效
- 节奏是否合适
- 高潮/转折力道是否足够
- AI 标记项逐一确认

### 步骤 3：篇级检查

读取 `checklists/part-check.md`，按清单逐项检查：

1. 通读本篇各章，确认情节线收束
2. 对照 outline 弧光规划表，检查里程碑是否按时到达
3. 检查节奏曲线
4. 检查角色演变连贯性（对照 characters/*.md 演变表）
5. 检查伏笔埋设/回收平衡

### 步骤 4：卷级检查

读取 `checklists/volume-check.md`，按清单逐项检查：

1. 主线推进幅度合理
2. 各篇节奏张弛有度
3. 卷末有高潮或转折
4. 全卷角色演变连贯，无跳跃
5. 三条线变化无矛盾
6. 伏笔按计划回收，无遗漏
7. 设定无前后矛盾
8. 全卷风格稳定

### 步骤 5：生成检查报告

写入 `reports/volX/partY-chZZ-check.md`（章级）或 `reports/volX/partY-check.md`（篇级）或 `reports/volX-check.md`（卷级）。

**章级报告格式**：

```markdown
# 章级检查报告 — chZZ

## AI 自动核查

| 检查项 | 结果 | 判断 |
|--------|------|------|
| 本章总字数 | XXXX 字 | 通过 / 标记 |
| 各幕字数偏差 | ... | 通过 / 标记 |
| 章末钩子对照 | ... | 通过 / 标记 |
| 情绪走向 | ... | 通过 / 标记 |
| 最长连续对话 | XX 句 | 通过 / 标记 |
| 禁区扫描 | XX 处违规 | 通过 / 标记 |

## 作者确认项

- [ ] 章末钩子有效
- [ ] 节奏不拖不赶
- [ ] 高潮/转折力道足够
- [ ] 标记项已逐一确认

## 待处理项

（列出需要修改的具体问题和位置）
```

### 步骤 6：输出总结

```markdown
## 检查完成 — [chZZ / partY / volX]

- **检查层级**：章级 / 篇级 / 卷级
- **结论**：全部通过 / 有标记项（见报告）
- **报告路径**：reports/volX/...

## 需要处理的问题

1. ...
2. ...
```

## 禁止事项

- 不得跳过 AI 自动核查部分直接要求作者确认
- AI 只做计数和对比，不做审美判断（标记项需由作者确认）
- 不得修改正文——检查只生成报告，不修改内容
